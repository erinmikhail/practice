Пов: Для Артема.

Архитектурные правила: 
Единый префикс: Абсолютно все эндпоинты должны начинаться с /api/.
Английские ключи: Никаких русских слов в JSON. Строго income/expense и категории на латинице (groceries, transport и т. д.), как прописано в контракте Даниила.
Никакого SQL в эндпоинтах: Вы не пишете db.query(...). Для любых операций с базой вы импортируете готовые функции Миши из backend/database/crud.py и передаете им сессию БД (db: Session).
CORS: В main.py должен быть настроен CORSMiddleware с доступом для http://localhost:5173, иначе фронт Даниил выдаст ошибку сети.

К пятнице:
Зона ответственности: Файл main.py (сборка сервера) и папка api/endpoints/transactions.py.
Твоя задача — сделать так, чтобы форма добавления и таблица истории на фронтенде ожили.
GET /api/transactions * Вызывает функцию Миши get_user_transactions(db, user_id=1, limit=100).
Возвращает JSON-массив операций.
POST /api/transactions
Принимает JSON с полями: amount, type, category, date, comment.
Валидирует через Pydantic (схемы уже написаны).
Передает данные в функцию Миши create_transaction(db, transaction_data, user_id=1).
DELETE /api/transactions/{id}
Берет идентификатор из URL, передает в Мишину delete_transaction(db, transaction_id, user_id=1).



Потом доделать: 
Авторизация (JWT): Убрать хардкод user_id = 1. Написать эндпоинты /api/auth/register и /api/auth/login. Выдавать токен. Все эндпоинты транзакций закрыть зависимостью Depends(get_current_user).

Пагинация: Настроить передачу параметров ?skip=0&limit=50 из фронтенда в функции базы данных.

Аналитика: Написать эндпоинт GET /api/transactions/summary, который будет вызывать Мишину функцию get_finance_summary для расчета агрегатов на стороне базы (сейчас фронт считает это сам, но при 10 000 записей браузер зависает, считать должен сервер).
