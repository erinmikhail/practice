Полная краткая задача: 
Каркас и роутинг: Верстать index.html (или настроить React/Vue, если используется фреймворк). 
Сделать два основных экрана: «Главная» (список и статистика) и «Импорт чека».
Интеграция API (Axios): Написать JS-функции для запросов к бэкенду (GET /transactions, POST /transactions).
Drag & Drop и FormData: Реализовать зону загрузки картинок. 
Картинка должна быть правильно упакована в бинарный формат FormData для отправки на сервер.
Управление состоянием (экран черновиков): Самая сложная задача. Когда от бэкенда приходит ответ нейросети, должен отобразить эти транзакции в редактируемой таблице (в памяти браузера, без сохранения в базу). 
И только после нажатия кнопки «Подтвердить» отправить итоговый JSON на сервер.

К утру пятницы: 
Хотя бы одна страница (SPA)
SPA (Single Page Application) — это веб-приложение, которое загружает одну единственную HTML-страницу и динамически обновляет её содержимое по мере взаимодействия пользователя, без перезагрузки всей страницы.
Простая форма, таблица истории, кнопка загрузки скрина и экрана черновиков.

Форма - мое представление: 
Поле ввода суммы
выбор типа операции доход\расход
выбор даты
поле для комментария
кнопка добавить

Таблица истории: 
Дата, категория, тип операции, сумма, комментарий, действие.
