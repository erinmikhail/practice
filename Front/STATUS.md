# Конспект проекта — что сделано и почему

Учебный проект: веб-приложение для учёта личных финансов с AI-агентом для разбора банковских транзакций.
Команда — 5 человек. Моя (фронтенда) зона — весь UI. Бэкенд (FastAPI + SQLite) и AI-скрипты (GigaChat) делают другие ребята.

Полное ТЗ: см. исходный файл техзадания. Архитектура — `ARCHITECTURE.md` (UML-диаграмма компонентов).

## Стек

React + Vite + Tailwind v4 (CSS-first, без `tailwind.config.js`) + `react-router-dom`. Обычный JavaScript, без TypeScript — по договорённости, чтобы проще было разбираться (я начинающий разработчик).

## Что готово

- **Главная** (`/`) — карточки Доходы/Расходы/Баланс + круговая диаграмма расходов по категориям.
- **Операции** (`/operations`) — форма добавления (с валидацией), список операций с удалением, фильтр **Все / Доходы / Расходы**.
- **Импорт (AI)** (`/import`) — вкладки **Текст** и **Скриншот** (вкладку «Файл» убрали по решению команды — усложняла без пользы). Разбор показывает превью операций, пользователь подтверждает перед добавлением в реальный список.
- Данные хранятся в `localStorage` браузера — переживают перезагрузку страницы.

## Как устроены данные (важно для будущего подключения бэка)

Всё идёт через один файл — `src/api/operations.js`. Он сейчас работает через:
- `localStorageAdapter.js` — реальное хранилище (пока бэка нет),
- `aiMockParser.js` — заглушка AI-разбора (задержка + эвристика по ключевым словам для текста, статичные демо-данные для скриншота).

Когда бэкенд поднимет эндпоинты — поменяется **только этот один файл** (переключится на `client.js`, уже настроенный axios-клиент). Компоненты вообще не знают, откуда берутся данные — это специально.

## Договорённость с бэкенд-командой

Ребята прислали `text_AI.py` и `OCR_AI.py` (GigaChat + Tesseract OCR) — они лежат в `backend-scripts/`. Мы сверили формат: категории теперь общие slug'и (`groceries`, `transport`, `cafe`, `entertainment`, `health`, `transfers`, `salary`, `other`), оба скрипта возвращают массив операций одинаковой формы `{amount, date, type, category, comment}`.

Полный контракт для HTTP-обёртки (пути, JSON, ошибки, CORS) — в `backend-scripts/API_CONTRACT.md`. Бэкендерам осталось обернуть эти функции в FastAPI-эндпоинты `POST /api/import/text` и `POST /api/import/image`.

## Заметки/грабли, чтобы не наступать снова

- **Не используем `recharts`** — версия 3.9.1 рисовала диаграмму неправильно (крошечный сектор вместо целого круга). Заменили на свою диаграмму на CSS `conic-gradient` — без зависимостей, работает надёжно.
- Домашняя папка пользователя (`~`) сама является git-репозиторием — если понадобится `git init` внутри этого проекта, сначала стоит решить, не будет ли это вложенным репозиторием (возможно, стоит вынести проект из `~`).

## Дальше (не сделано)

- Дождаться от бэкенда реальных эндпоинтов `/api/operations` и `/api/import/*` и переключить `operations.js` на `client.js`.
- Возможно, добавить редактирование операций (сейчас только добавление/удаление) — в ТЗ явно не требуется, но может пригодиться.
