# Конспект проекта — что сделано и почему

Учебный проект: веб-приложение для учёта личных финансов с AI-агентом для разбора банковских транзакций.
Команда — 5 человек. Моя (фронтенда) зона — весь UI. Бэкенд (FastAPI + SQLite) и AI-скрипты (GigaChat) делают другие ребята.

## Стек

React + Vite + Tailwind v4 (CSS-first, без `tailwind.config.js`) + `react-router-dom`. Обычный JavaScript, без TypeScript.

## Что готово

- **Главная** (`/`) — карточки Доходы/Расходы/Баланс + круговая диаграмма расходов по категориям.
- **Операции** (`/operations`) — форма добавления (с валидацией), список операций с удалением, фильтр **Все / Доходы / Расходы**.
- **Импорт (AI)** (`/import`) — вкладки **Текст** и **Скриншот**. Разбор показывает превью операций (черновики), пользователь подтверждает перед добавлением в реальный список.

## Архитектура — специально максимально простая

Никакого Context, никаких кастомных хуков и слоёв-адаптеров — это была лишняя сложность, пока не было настоящего бэкенда. Теперь бэкенд есть, и всё устроено так:

- **`src/api.js`** — единственный файл, который ходит на сервер (через axios). Пять простых функций: `getOperations`, `createOperation`, `deleteOperation`, `importFromText`, `importFromImage`. Больше нигде в коде axios не используется.
- **`src/App.jsx`** — здесь и только здесь хранится список операций (`useState` + `useEffect`, без Context). Все три страницы получают его обычными пропсами через `<Route element={...} />`.
- **`src/utils/stats.js`** — просто функция `calculateStats(operations)`, которая на каждый рендер пересчитывает доходы/расходы/баланс/разбивку по категориям. Никакого кэширования — операций мало, пересчитывать дёшево, а объяснять проще некуда.
- Компоненты (`OperationForm`, `OperationList`, `AiImportPanel` и т.д.) — просто получают данные и колбэки через пропсы, ничего не знают о сервере.

Итого: чтобы понять, откуда берутся данные — нужно прочитать один файл (`api.js`) и один компонент (`App.jsx`). Больше никаких "слоёв абстракции".

## Подключение к бэкенду

Бэкенд поднят командой на FastAPI (`/api/transactions` — CRUD операций, `/api/import/text` и `/api/import/image` — AI-разбор). Адрес бэкенда берётся из `.env` (`VITE_API_BASE_URL`, по умолчанию `http://localhost:8000/api`).

Заглушки (localStorage, фейковый AI-парсер) убраны полностью — фронт больше не работает без бэкенда.

## Заметки/грабли, чтобы не наступать снова

- **Не используем `recharts`** — версия 3.9.1 рисовала диаграмму неправильно (крошечный сектор вместо целого круга). Заменили на свою диаграмму на CSS `conic-gradient` — без зависимостей, работает надёжно.
- У `/api/transactions/` на бэке есть завершающий слэш (так зарегистрирован роут в FastAPI) — в `api.js` он учтён, если будете писать новые запросы к этому ресурсу, не забывайте про него.

## Дальше (не сделано)

- Авторизация на бэке пока не готова, `user_id=1` захардкожен как заглушка — фронта это пока не касается.
- Возможно, добавить редактирование операций (сейчас только добавление/удаление) — в ТЗ явно не требуется, но может пригодиться.
