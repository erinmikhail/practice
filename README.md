# Практика

#### Инструкция по запуску и работе с докером

---
1. Скачайте и установите Docker Desktop для вашей ОС (Windows/Mac/Linux) с официального сайта.
2. Запустите Docker Desktop и убедитесь, что он работает. В левом нижнем углу программы должен гореть зеленый статус Engine running (иконка кита в трее возле часов должна перестать мигать).
3. Установите в свой VS Code официальное расширение Dev Containers от Microsoft (ищите его на вкладке расширений слева).
4. Клонируйте наш репозиторий с GitHub на свой компьютер и откройте папку проекта в VS Code. открывайте именно корневую папку проекта
5. Потом Нажать `Ctrl + Shift + P`
6. Вводите `Dev Containers: Reopen and ...` (и что-то дальше не помню). Ждете загрузку и все готово.

#### Архитектура проекта
---

practice/
│
├── .devcontainer/                Слой инфраструктуры разработки
│   ├── devcontainer.json         Конфигурация расширений и портов VS Code
│   └── Dockerfile                Инструкция сборки ОС контейнера (Python, SQLite)
│
├── frontend/                     ИЗОЛИРОВАННЫЙ СЛОЙ КЛИЕНТА (Фронтенд)
│   ├── static/                   Статические ресурсы
│   │   ├── css/
│   │   │   └── style.css         Стили и верстка интерфейса
│   │   └── js/
│   │       └── app.js            Логика (Axios-запросы, Drag&Drop, State)
│   └── templates/                HTML-каркас
│       └── index.html            Single Page Application (SPA) страница
│
├── backend/                      ИЗОЛИРОВАННЫЙ СЛОЙ СЕРВЕРА (Бэкенд на Python)
│   ├── __init__.py
│   ├── main.py                   Точка входа: инициализация FastAPI, Middleware (CORS)
│   ├── config.py                 Переменные окружения, пути к БД, API-ключи нейросетей
│   │
│   ├── database/                 ВСЁ ЧТО СВЯЗАНО С БАЗОЙ ДАННЫХ
│   │   ├── __init__.py
│   │   ├── session.py            Настройка Engine SQLAlchemy и генератор сессий подключения
│   │   ├── models.py             Декларативные модели (структура таблиц SQLite на диске)
│   │   └── crud.py               Чистые SQL/ORM запросы (создание, чтение, апдейты, аналитика)
│   │
│   ├── api/                      СЛОЙ СЕТЕВЫХ ЭНДПОИНТОВ (Контроллеры / Роуты)
│   │   ├── __init__.py
│   │   ├── deps.py               Зависимости для инжекта (проверка авторизации, выдача сессий БД)
│   │   ├── schemas.py            Валидационные схемы Pydantic (контракты для запросов/ответов)
│   │   └── endpoints/            Конкретные группы HTTP-маршрутов
│   │       ├── __init__.py
│   │       ├── auth.py           Логика пользователей: регистрация, вход, JWT-токены
│   │       ├── transactions.py   Логика транзакций: CRUD ручных операций и графики статистики
│   │       └── ai.py             Маршрут для обработки AI: прием картинок/текста чеков
│   │
│   └── services/                 СЛОЙ СЛОЖНОЙ БИЗНЕС-ЛОГИКИ (Сервисы)
│       ├── __init__.py
│       └── ai_agent.py           Модуль AI-агента (сборка промптов, Vision API, парсинг JSON)
│
├── requirements.txt              Все зависимости Python-пакетов проекта
└── .gitignore                    Защита Git от попадания мусора, файлов БД и секретных ключей
