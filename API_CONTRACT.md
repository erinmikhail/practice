# Контракт API для AI-импорта операций

Фронт (React) уже реализован и ждёт эти два эндпоинта. Ниже — то, что нужно обернуть вокруг `text_AI.py` (`process_user_text`) и `OCR_AI.py` (`process_image`) в FastAPI, чтобы фронт мог их вызвать.

Базовый адрес фронт берёт из `VITE_API_BASE_URL` (сейчас `http://localhost:8000/api`), так что оба пути ниже — относительно него.

---

## 1. Импорт из текста

```
POST /api/import/text
Content-Type: application/json
```

**Тело запроса:**
```json
{ "text": "купил кофе 250\nзарплата 50000 сегодня" }
```

**Ответ 200 OK** — массив транзакций (даже если она одна):
```json
[
  {
    "amount": 250,
    "date": "2026-07-01",
    "type": "expense",
    "category": "cafe",
    "comment": "купил кофе 250"
  },
  {
    "amount": 50000,
    "date": "2026-07-01",
    "type": "income",
    "category": "salary",
    "comment": "зарплата 50000 сегодня"
  }
]
```

Это ровно то, что уже возвращает `process_user_text(user_input)` — просто вызвать её с `text` из тела запроса и отдать результат как JSON.

Если распознать ничего не удалось — вернуть `[]` (пустой массив), а не ошибку. Фронт покажет пользователю "не удалось найти операции".

---

## 2. Импорт из скриншота

```
POST /api/import/image
Content-Type: multipart/form-data
```

**Тело запроса:** одно поле `file` с картинкой (`image/*`).

**Ответ 200 OK** — тот же формат массива транзакций, что и у текста (см. выше). Это то, что уже возвращает `process_image(image_path)` — на бэке нужно сохранить загруженный файл во временный путь (или передать в Pillow напрямую через `Image.open(file.stream)`, не обязательно писать на диск) и вызвать функцию.

---

## Общее для обоих эндпоинтов

- **Поле `category`** — строго одно из: `groceries`, `transport`, `cafe`, `entertainment`, `health`, `transfers`, `salary`, `other`. Скрипты уже приводят к этому списку (`if t.get('category') not in CATEGORIES_KEYS: t['category'] = 'other'`) — ничего дополнительно готовить не нужно.
- **Поле `type`** — строго `"income"` или `"expense"`.
- **Поле `date`** — строка `"YYYY-MM-DD"`.
- **Поле `amount`** — число (`float`), без знака и без валюты.
- **Поле `comment`** — строка, может быть пустой.
- Ответ **не должен** содержать `id` — фронт сам сгенерирует id и покажет операции как превью для подтверждения пользователем перед сохранением.

## Ошибки

Если GigaChat недоступен или упал (см. `except Exception as e` в скриптах) — вернуть **502 Bad Gateway** с телом `{ "error": "..." }`, а не 500 с трейсбеком. Фронт покажет пользователю сообщение об ошибке импорта.

## Безопасность

`GIGACHAT_AUTH_KEY` должен читаться из `.env` **только на сервере** (как сейчас через `load_dotenv()` в скриптах) и никогда не передаваться на фронт ни в одном ответе.

## CORS

Пока фронт крутится на `http://localhost:5173` (Vite dev server), а бэк — на `http://localhost:8000`, нужно включить CORS для этого origin (`fastapi.middleware.cors.CORSMiddleware`, `allow_origins=["http://localhost:5173"]`), иначе браузер заблокирует запросы.

## Как это ляжет во фронт

Когда эндпоинты будут готовы, в файле `src/api/operations.js` в функции `importFromAI` мок-вызовы (`aiMockParser.parseText/parseImage`) заменятся на:
```js
await apiClient.post('/import/text', { text: payload.text })
await apiClient.post('/import/image', formData) // FormData с payload.file
```
Больше нигде в коде фронта менять ничего не придётся — вся остальная логика (превью, подтверждение, добавление в список) уже рассчитана на этот формат ответа.
